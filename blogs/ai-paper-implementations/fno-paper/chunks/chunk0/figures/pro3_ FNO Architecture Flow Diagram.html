INPUT
Input Function a(x)
Initial condition / Parameters
▼
What it represents:
  • Initial conditions of the PDE (e.g., initial temperature distribution)
  • Boundary conditions
  • PDE coefficients or forcing terms

Shape: (batch, x, y, d) where d is input channels

LAYER 1
Lifting Layer
Pointwise MLP: ℝᵈ → ℝʷⁱᵈᵗʰ
▼
Purpose: Lift input from d dimensions to a higher dimensional representation (width).
  • Typically a 2-layer MLP applied pointwise
  • Common width: 32, 64, or 128 channels
  • Enables the network to learn richer representations

Code: nn.Linear(d, width)

ℱ
Fourier Layer 1
FFT → Linear → iFFT + Bypass
▼
Spectral Convolution:
  • FFT: Transform to frequency domain
  • Linear: Multiply with learnable weights R
  • iFFT: Transform back to spatial domain
  • Bypass: Add local linear transform W·x

Formula: σ(Wv + K(v)) where K is spectral conv

ℱ
Fourier Layer 2
FFT → Linear → iFFT + Bypass
▼
Key insight: Convolution in spatial domain = multiplication in frequency domain.
  • Much more efficient for global operations
  • Captures long-range dependencies
  • Only keeps first k Fourier modes (truncation)
ℱ
Fourier Layer 3
FFT → Linear → iFFT + Bypass
▼
Mode truncation: Only k_max lowest frequency modes are kept.
  • Higher modes often represent noise
  • Reduces computational cost significantly
  • Typical: k_max = 12 to 20 modes
ℱ
Fourier Layer 4
FFT → Linear → iFFT + Bypass
▼
Activation: GELU or ReLU applied after each layer.
  • Non-linearity enables learning complex mappings
  • Applied in spatial domain (after iFFT)
  • Final layer often has no activation
×L layers (typically 4)
FINAL LAYER
Projection Layer
Pointwise MLP: ℝʷⁱᵈᵗʰ → ℝᵒᵘᵗ
▼
Purpose: Project from high-dimensional space back to output dimension.
  • Typically a 2-layer MLP applied pointwise
  • Maps width channels to output channels (e.g., 1 for scalar field)
  • No activation on final output

Code: nn.Linear(width, out)

OUTPUT
Solution u(x)
PDE Solution
▼
What it represents:
  • The predicted solution to the PDE
  • Evaluated at the same grid points as input
  • Can be at any resolution (resolution-invariant!)

Key advantage: Once trained, inference takes only milliseconds!
